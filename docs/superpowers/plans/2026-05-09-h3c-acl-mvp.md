# H3C ACL MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development or superpowers:executing-plans to
> implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for
> tracking.

**Goal:** Add isolated H3C numeric IPv4 advanced ACL create/read/verify support
through the transactional NETCONF path.

**Safety boundary:** Never use an existing ACL id in real-device acceptance.
Read existing ACL ids first, choose an absent candidate, re-check it immediately
before write, and delete only that candidate during cleanup. Do not bind the ACL
to any production feature in this MVP.

**Tech Stack:** Rust model/proto/mapper/diff, Python adapter renderer/parser,
pytest, GitHub Actions.
